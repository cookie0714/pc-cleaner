# Mac向け「ゴミよおさらば」機能仕様

## 1. アプリ概要

Mac のローカルファイルをスキャンし、不要ファイル候補をカテゴリ別に提示して、ユーザー操作により安全に削除するデスクトップアプリ。

初期仕様では、完全削除ではなく macOS のゴミ箱へ移動することを標準動作とする。

## 2. 画面構成

### 2.1 ホーム画面

表示内容:

- Mac のストレージ空き容量
- 前回スキャン日時
- 前回検出サイズ
- 前回削除サイズ
- スキャン開始ボタン
- 主要カテゴリの概要

操作:

- 「スキャン開始」でスキャン画面へ遷移する
- 「履歴を見る」で履歴画面へ遷移する
- 「設定」で設定画面へ遷移する

### 2.2 スキャン画面

表示内容:

- スキャン進捗
- 現在処理中のカテゴリ
- 検出済みサイズ
- 検出済みファイル数
- キャンセルボタン

動作:

- スキャン開始後、カテゴリごとに順次処理する
- 結果が得られたカテゴリから画面に反映する
- キャンセル時は途中結果を破棄するか、確認のうえ結果画面へ進む

### 2.3 結果画面

表示内容:

- 検出合計サイズ
- 検出ファイル数
- カテゴリ別カードまたはリスト
- 各カテゴリの選択チェック
- 削除予定サイズ
- 削除確認ボタン

カテゴリ表示項目:

- カテゴリ名
- 説明
- 合計サイズ
- ファイル数
- リスクレベル
- 推奨選択状態

### 2.4 詳細画面

表示内容:

- ファイルパス
- サイズ
- 最終更新日時
- 最終アクセス日時
- 所属カテゴリ
- 選択チェック
- Finder で表示する操作

並び替え:

- サイズ順
- 更新日時順
- パス順

フィルタ:

- 選択済みのみ
- 未選択のみ
- 指定サイズ以上
- 指定日数より古い

### 2.5 システムデータ画面

macOS のストレージ表示に現れる「システムデータ」に相当する領域を、読み取り専用で集計する。

表示内容:

- 読み取れた範囲のシステムデータ相当領域の検出合計サイズ
- カテゴリ別サイズ内訳
- カテゴリごとのスキャン対象ルート
- カテゴリ内の上位項目、パス、サイズ、更新日時、読み取り状態
- 読み取れなかった項目、サイズ不明である旨、理由

操作:

- 「内訳をスキャン」で読み取り専用スキャンを実行する
- 項目を Finder で表示する
- サイズ、更新日時、パスで並び替える

備考:

- ファイル本文は読み取らず、パス、メタデータ、サイズのみを集計する。
- 削除候補には含めず、結果画面の選択状態や削除確認には接続しない。
- 書類、写真、メール、デベロッパ、iCloud Drive など macOS が別カテゴリで表示するユーザーデータは対象外にする。
- フルディスクアクセスがない場合、アクセスできない領域はスキップ理由を表示する。
- APFS スナップショットなど、通常のファイルツリーで検出できない領域は今後の拡張対象とする。
- ファイルツリー上で検出できたが読み取れない項目は、サイズ不明として内訳に表示する。

### 2.6 書類データ画面

macOS のストレージ表示に現れる「書類」に相当するユーザーデータを、読み取り専用で集計する。

表示内容:

- 読み取れた範囲の書類データの検出合計サイズ
- 場所別サイズ内訳
- 場所ごとのスキャン対象ルート
- 場所内の上位項目、パス、サイズ、更新日時、読み取り状態
- 読み取れなかった項目、サイズ不明である旨、理由

対象場所:

- `~/Documents`
- `~/Desktop`
- `~/Downloads`
- iCloud Drive のローカル同期領域
- 共有ユーザー領域と `~/Public`

操作:

- 「内訳をスキャン」で読み取り専用スキャンを実行する
- 項目を Finder で表示する
- サイズ、更新日時、パスで並び替える

備考:

- ファイル本文は読み取らず、パス、メタデータ、サイズのみを集計する。
- 削除候補には含めず、結果画面の選択状態や削除確認には接続しない。
- 写真、メール、音楽、動画など別カテゴリで扱うべきデータは対象外にする。
- iCloud 上にのみ存在するプレースホルダなど、サイズを確認できない項目はサイズ不明として表示する。

### 2.7 削除確認画面

表示内容:

- 削除予定カテゴリ
- 削除予定ファイル数
- 削除予定合計サイズ
- ゴミ箱へ移動する旨の説明
- 注意が必要なカテゴリの警告

操作:

- 「ゴミ箱へ移動」で削除を実行する
- 「戻る」で結果画面に戻る

### 2.8 削除完了画面

表示内容:

- 削除成功サイズ
- 削除成功件数
- 失敗件数
- 失敗理由一覧
- ゴミ箱を開くボタン

### 2.9 履歴画面

表示内容:

- 実行日時
- 削除サイズ
- 削除件数
- 対象カテゴリ
- 実行結果

### 2.10 設定画面

設定項目:

- スキャン対象カテゴリ
- 除外パス
- 古いファイル判定の日数
- 最小表示サイズ
- 削除方法
- 履歴保持期間

## 3. スキャンカテゴリ仕様

### 3.1 ユーザーキャッシュ

対象例:

- `~/Library/Caches`

初期選択:

- オン

リスク:

- 低

備考:

- アプリが再生成する一時データを想定する。ただし実行中アプリのキャッシュ削除は失敗する場合がある。

### 3.2 アプリケーションログ

対象例:

- `~/Library/Logs`

初期選択:

- オン

リスク:

- 低

備考:

- トラブル調査中のログが必要な可能性があるため、詳細画面で確認可能にする。

### 3.3 一時ファイル

対象例:

- ユーザー領域内の一時ディレクトリ
- アプリが作成した明確な一時ファイル

初期選択:

- オン

リスク:

- 中

備考:

- 実行中プロセスが使用しているファイルは削除対象から外す。

### 3.4 ダウンロード内の古いファイル

対象例:

- `~/Downloads`

初期選択:

- オフ

リスク:

- 中

備考:

- ユーザー作成物や重要ファイルが含まれる可能性があるため、自動選択しない。
- デフォルトでは 90 日以上更新されていないファイルを候補にする。

### 3.5 ゴミ箱

対象例:

- `~/.Trash`

初期選択:

- オン

リスク:

- 中

備考:

- ゴミ箱内ファイルの完全削除は初期リリースでは確認ステップを追加する。

### 3.6 ブラウザキャッシュ

対象例:

- Safari、Chrome、Firefox などのキャッシュ領域

初期選択:

- オン

リスク:

- 低から中

備考:

- ブラウザが起動中の場合、削除失敗や再生成が発生する可能性がある。
- Cookie、ログイン情報、履歴は初期リリースでは削除対象外にする。

### 3.7 開発者向けファイル

対象例:

- Xcode DerivedData
- シミュレータの一時データ
- パッケージマネージャーのキャッシュ

初期選択:

- オフ

リスク:

- 中

備考:

- 開発環境に影響する可能性があるため、カテゴリ説明を明確にする。

## 5. 削除ルール

### 5.1 基本方針

- 初期動作はゴミ箱へ移動とする
- 完全削除は設定で明示的に有効化された場合のみ表示する
- 削除前に必ず確認画面を表示する
- 保護パスに該当するファイルは削除対象にしない

### 5.2 保護パス

以下は削除対象から除外する:

- `/System`
- `/Library` のうち OS 管理領域
- `/Applications`
- `~/Applications`
- `~/Documents`
- `~/Desktop`
- `~/Pictures`
- `~/Movies`
- `~/Music`
- `.git` ディレクトリ
- プロジェクトルート内のソースファイル

### 5.3 ファイル使用中の扱い

- 使用中またはロック中のファイルはスキップする
- スキップ理由を削除完了画面に表示する
- 強制削除は行わない

## 6. 権限仕様

### 6.1 通常権限

通常のユーザー領域に対しては、可能な範囲でスキャンを実行する。

### 6.2 フルディスクアクセス

一部領域のスキャンに必要な場合のみ、フルディスクアクセスの必要性を表示する。

表示内容:

- なぜ必要か
- 許可しない場合に使える機能
- システム設定を開く導線

### 6.3 管理者権限

初期リリースでは管理者権限が必要な削除は対象外とする。

## 7. エラー仕様

| エラー | 表示方針 |
| --- | --- |
| 権限不足 | 必要な権限と影響範囲を表示する |
| ファイルが存在しない | スキャン後に消えた可能性として処理を継続する |
| ファイル使用中 | スキップし、削除完了画面に表示する |
| ゴミ箱移動失敗 | 理由を表示し、再試行可能にする |
| スキャン中断 | 中断済みとしてホームへ戻す |

## 8. 技術方針

### 8.1 採用技術

デスクトップアプリの実装基盤は Tauri を採用する。

想定構成:

- Tauri
- Rust
- TypeScript
- フロントエンド UI フレームワーク

Tauri 側の Rust コードでは、ファイルスキャン、サイズ集計、ゴミ箱への移動、権限状態の確認など、OS やファイルシステムに近い処理を担当する。

フロントエンド側では、画面表示、ユーザー操作、選択状態、スキャン結果の可視化、設定画面を担当する。

初期リリースの配布対象は Apple Silicon Mac のみとし、ビルド・動作確認・パフォーマンス検証は `arm64` を前提に行う。

### 8.2 アーキテクチャ方針

- UI 層、スキャン層、削除層、履歴保存層を分離する
- Tauri コマンドを UI とネイティブ処理の境界として扱う
- スキャンカテゴリはプラグイン的に追加できる構造にする
- 削除処理は共通の安全チェックを必ず通す
- ファイル操作はテスト可能なインターフェース越しに実装する

### 8.3 ローカル保存

保存対象:

- 設定
- 除外パス
- スキャン履歴
- 削除履歴

保存しない対象:

- 不要ファイルの全文内容
- 外部送信用のファイルパス情報

## 9. テスト方針

### 9.1 単体テスト

- カテゴリ判定
- 保護パス判定
- サイズ集計
- 選択状態の集計
- 削除前チェック

### 9.2 結合テスト

- スキャンから結果表示まで
- 結果選択から削除確認まで
- ゴミ箱移動の成功/失敗
- 権限不足時の表示

### 9.3 手動テスト

- macOS の権限設定変更後の挙動
- 大量ファイル環境でのスキャン
- ブラウザ起動中のキャッシュ削除
- Finder で表示する導線
- ゴミ箱を開く導線

## 10. MVP

最小実用版では以下を実装する:

1. ホーム画面
2. ユーザーキャッシュ、ログ、ゴミ箱のスキャン
3. カテゴリ別結果表示
4. ファイル詳細表示
5. ゴミ箱への移動
6. 削除履歴
7. 除外パス設定
