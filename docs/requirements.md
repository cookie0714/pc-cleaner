# Mac向け「ゴミよおさらば」要件定義

## 1. 目的

Mac 上に蓄積された不要ファイルや容量を圧迫しているデータを、ユーザーが安全に把握・削除できるデスクトップアプリを提供する。

このアプリは「自動で何でも削除する」ことではなく、ユーザーが削除対象を理解し、確認したうえで整理できることを重視する。

## 2. 想定ユーザー

- Mac のストレージ容量不足に困っている一般ユーザー
- キャッシュ、ログ、ダウンロード済みファイルなどを手軽に整理したいユーザー
- Finder やターミナルでの手動整理に不安があるユーザー
- アプリ開発、デザイン、動画編集などで一時ファイルが増えやすいユーザー

## 3. 対象プラットフォーム

- macOS 向けデスクトップアプリ
- Apple Silicon Mac のみを対象とする
- 初期リリースでは macOS 13 Ventura 以降を主対象とする
- アプリ基盤は Tauri を採用する

## 4. 提供価値

- ストレージ使用状況をわかりやすく可視化する
- 削除しても比較的安全な候補を分類して提示する
- 削除前に内容、サイズ、影響範囲を確認できる
- 誤削除のリスクを下げるため、保護ルールと復元手段を用意する
- Mac の権限モデルに沿って、必要なアクセス許可を明示する

## 5. スコープ

### 5.1 初期リリースで対応する範囲

- ストレージ使用量のスキャン
- 不要ファイル候補の分類表示
- 削除対象の選択
- 削除前の確認画面
- ゴミ箱への移動による安全な削除
- スキャン結果のサイズ順・種類別表示
- 最終スキャン日時、削除履歴の表示

### 5.2 初期リリースで対象とする削除候補

- ユーザーキャッシュ
- アプリケーションキャッシュ
- ログファイル
- 一時ファイル
- ダウンロードフォルダ内の古いファイル
- ゴミ箱内のファイル
- ブラウザキャッシュ
- Xcode や開発ツールの派生データなど、開発者向け大容量ファイル

### 5.3 初期リリースでは対象外

- ウイルス検知・マルウェア除去
- メモリ解放を目的にした常駐最適化
- カーネル拡張や低レベルなシステム変更
- ユーザー確認なしの自動削除
- 他社アプリの設定ファイルやユーザーデータの削除
- 重複写真や類似画像の高度な判定

## 6. 機能要件

### 6.1 スキャン

- ユーザーが明示的にスキャンを開始できる
- スキャン対象をカテゴリ単位で選択できる
- スキャン中の進捗、現在処理中のカテゴリ、推定検出サイズを表示する
- スキャンはキャンセルできる
- 権限不足で読めない領域がある場合、その理由と必要な許可を表示する

### 6.2 分類

- 検出したファイルをカテゴリ別に分類する
- 各カテゴリごとに件数、合計サイズ、リスクレベルを表示する
- ファイル単位でパス、サイズ、最終更新日時を確認できる
- 削除推奨度を「安全」「確認推奨」「慎重に確認」のように段階表示する

### 6.3 削除

- 削除前に必ず確認画面を表示する
- 初期設定では完全削除ではなくゴミ箱へ移動する
- 削除対象の合計サイズ、件数、カテゴリを確認できる
- 削除できなかったファイルがある場合、理由と再試行可否を表示する
- 管理者権限が必要な削除は初期リリースでは原則扱わない

### 6.4 履歴

- 実行日時、削除サイズ、削除件数を履歴として表示する
- 履歴から過去の削除対象カテゴリを確認できる
- ゴミ箱へ移動したファイルについて、Finder でゴミ箱を開ける導線を用意する

### 6.5 設定

- スキャン対象カテゴリのオン・オフを設定できる
- 除外パスを登録できる
- 指定サイズ未満のファイルを非表示にできる
- 古いファイル判定の日数を設定できる
- 完全削除機能は初期状態で無効とし、導入する場合は明確な警告を表示する

## 7. 非機能要件

### 7.1 安全性

- ユーザーのドキュメント、写真、音楽、動画、プロジェクトファイルを自動選択しない
- システム領域やアプリ本体を削除対象にしない
- 削除前にユーザーが対象を確認できる UI を必ず用意する
- 既知の危険パスを保護リストとして持つ

### 7.2 パフォーマンス

- スキャン中も UI が固まらない
- 大量ファイルを扱う場合は段階的に結果を表示する
- ファイルサイズ計算はバックグラウンドで行う
- スキャン結果は必要に応じてキャッシュし、再スキャン時の体感速度を改善する

### 7.3 プライバシー

- ファイル名やパスを外部送信しない
- 初期リリースではクラウド同期や分析送信を行わない
- クラッシュレポート等を導入する場合は、ユーザーに明示して同意を得る

### 7.4 権限

- 必要なタイミングでのみ macOS の権限要求を行う
- フルディスクアクセスが必要な場合、理由と設定手順をアプリ内で説明する
- 権限がない場合も、アクセス可能な範囲で機能する

### 7.5 保守性

- 削除対象カテゴリは追加・変更しやすい構造にする
- スキャンロジックと UI 表示を分離する
- 削除ルールはテスト可能な形で管理する

## 8. UI 要件

- ホーム画面で現在の空き容量と前回スキャン結果を確認できる
- スキャン開始ボタンを明確に配置する
- 検出結果はカテゴリ別に一覧表示する
- カテゴリを開くとファイル単位の詳細を確認できる
- 削除前確認はモーダルまたは専用画面で表示する
- 危険度が高い操作は色、文言、確認ステップで明確に区別する

## 9. 成功指標

- 初回スキャン完了率
- スキャン後にユーザーが削除対象を理解できた割合
- 削除失敗率
- 誤削除に関する問い合わせ件数
- アプリ起動からスキャン開始までの所要時間
- スキャン完了までの所要時間

## 10. リスクと対策

| リスク | 対策 |
| --- | --- |
| 重要ファイルの誤削除 | 保護パス、削除前確認、ゴミ箱移動を標準にする |
| macOS 権限不足でスキャンできない | 権限状態を表示し、必要な許可だけ案内する |
| スキャンが重くなる | バックグラウンド処理、キャンセル、段階表示を実装する |
| ユーザーが削除内容を理解できない | カテゴリ説明、リスク表示、詳細表示を用意する |
| クリーナーアプリへの不信感 | 外部送信なし、過剰な警告なし、透明な削除ルールを明示する |
